# 2026-07-23 — 订阅更新假节点清洗误伤

## 现象

机场 dashboard 节点看起来比本地 yaml 多；每日 04:00 订阅任务曾 **`-t` 失败并回滚**。

## 原因

1. 机场 yaml 会插入 3 个假节点：`Expire:` / `Traffic:` / `Sync:`（仅展示用）。
2. `post_sub_clean.sh` 用 `/Expire:|Traffic:|Sync:/ { next }` 删行时，**策略组整行**也含这些字符串 → 连 `手动选择` 等分组定义一起被删 → 校验失败回滚。
3. Dashboard 计数通常含这 3 个假节点，故比清洗后的可用节点多。

## 修复

只删 `name: 'Expire:…'` 等假节点定义，再用 `sed` 从 proxy-groups 列表去掉残留引用。已在 A 上验证可 `-t` 通过并完成一次可用订阅更新。

真实节点数与机场大体一致（今日约 91；机场已去掉「韩国 02」属正常变动）。
